# Article Template

Use this template when composing term entries.

## Frontmatter

```mdx
---
title: {Term Name} ({Abbreviation if common})
summary: {Single sentence: what it is and why it matters. Plain language.}
tags:
  - {primary keyword}
  - {secondary keyword}
  - {related process}
  - {industry keyword}
categories:
  - {Primary category from existing set}
jurisdictions:
  - {List only if region-specific; empty [] if universal}
aliases:
  - {Common synonym}
  - {Abbreviation}
  - {Regional variant name}
related:
  - {slug-of-related-term-1}
  - {slug-of-related-term-2}
  - {slug-of-related-term-3}
---
```

## Template Notes

- **Sections are guidelines, not rigid.** Omit sections that don't apply. Add sections that are needed (e.g., "Pricing models" for a software tool, "Calculation methods" for financial concepts).
- **TLDR is mandatory** for every entry.
- **External references is mandatory** with at least 3 links.
- **Practical example is mandatory** for any concept involving calculations, processes, or workflows.
- Use `---` horizontal rules between major sections.
- Target **150-400 lines** for comprehensive entries. Quick entries can be shorter.
- No em-dashes. Use `--` double hyphen throughout.
